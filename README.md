# ggtypt-enhancement

南京大学智慧场馆预约辅助 userscript。脚本以浏览器本地面板的形式运行，提供场馆筛选、定时轮询、验证码辅助和订单流程辅助等功能。

## 安装

1. 安装 Tampermonkey 或兼容的 userscript 管理器。
2. 导入 `ggtypt-reservation-helper.user.js`。
3. 打开 `https://ggtypt.nju.edu.cn/venue/` 相关页面。

## 主要功能

- 总开关：控制所有自动流程。关闭时不会接管手动操作。
- 校区 / 场馆 / 运动项目：选择目标预约项目，脚本会自动推导对应预约页。
- 场馆结构：内置完整场馆项目目录，并在每次允许请求的页面加载时从 `website/venues` 接口更新合并。
- 场地 / 时段结构：内置 37 个项目的实际场地与时段快照；进入具体项目后再以 `reservation/day/info` 返回的数据精确覆盖并保存到本地，避免混入其他项目的场地或不存在的时段。
- 场地 / 时段：选择期望的场地号和时间段；不选则表示不限。
- 触发时间：设置定时开始轮询的时间；`00:00-08:00` 不发起自动请求，早于 `08:00:00` 的任务会排队到允许窗口。
- 定时恢复保护：定时目标保存在本地；页面恢复或重载后会重新校准。若浏览器冻结导致迟到超过 30 秒，本次任务会停止并记录错过窗口，不会在数小时后突然补跑。
- 轮询间隔：设置扫描频率，单位为毫秒。
- 最长轮询：达到设定分钟数后自动停止。
- 干跑模式：只扫描和记录命中结果，不自动点击场地。
- 主动刷新：通过预约接口刷新日期数据，辅助发现空位。
- 请求保护：主动 API 刷新最低间隔约 2 秒并带失败退避；纯网络连接错误在一分钟内最多恢复重试 5 次；检测到频繁访问限制后立即停止并进入冷却。
- 命中重载：接口发现空位但页面表格未更新时，自动重载预约页。
- 自动选同伴：提交前按设置选择同伴。
- 自动验证码：支持人工验证、本地自动、API 验证三种模式。
- API 点选反馈：API 请求和点击后的站点反馈分别最多等待 5 秒；超时后立即刷新剩余场地重试。
- 自动点支付：订单生成后尝试点击支付按钮。
- 随机候选：多个命中场地时随机选择一个。
- 单次预约：严格确认日期、时段、场地和订单标识并成功点击支付后，立即停止全部后续轮询和自动流程。
- 时段优先级：可拖动已选时段排序。实际命中时先选择最高优先级的可预约时段，再在该时段内选择场地。
- 定时任务空位判断：尚未支付时，从首次未发现目标时段空位起硬性计时 1 分钟，期间网络失败、轮询间断或页面重载不会重置；任何一次实际发现目标空位会清零。手动“立即轮询”不触发此自动停止。
- 停止：强制停止轮询、流程推进、验证码处理和支付重试。
- 清空日志：删除全部历史持久化日志，不影响预约设置。
- 诊断日志：流程卡住时保存页面地址、可见正文、弹窗、按钮和流程状态；不会保存密码或输入框内容。导出时按日期分段、缩进结构化数据并自动换行长页面文本；控件记录会标明来源、配置键和标签。空闲场地未命中时会逐项记录筛选原因。
- 最小化：点击标题栏减号将面板收起，点击右下角圆形按钮可居中恢复。
- 暗色模式：点击标题栏月亮或太阳图标切换主题。



## 验证码模式

- 人工验证：由用户手动完成验证码。
- 本地自动：根据内置程序计算，识别速度可能较慢。
- API 验证：验证码图片会发送至所配置的 API。

除 API 验证按用户配置发送验证码图片外，脚本设置、账号信息和日志均保存在本地。


## 使用建议

1. 首次使用保持干跑模式开启。
2. 选择校区、场馆、项目、场地和时段。
3. 点击扫描一次，确认日志中的命中结果符合预期。
4. 打开总开关。
5. 使用定时启动或立即轮询。
6. 确认流程稳定后，再关闭干跑模式。
