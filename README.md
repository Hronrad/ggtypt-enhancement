# ggtypt-enhancement

南京大学智慧场馆预约辅助 userscript。脚本以浏览器本地面板的形式运行，提供场馆筛选、定时轮询、验证码辅助和订单流程辅助等功能。

## 安装

1. 安装 Tampermonkey 或兼容的 userscript 管理器。
2. 导入 `ggtypt-reservation-helper.user.js`。
3. 打开 `https://ggtypt.nju.edu.cn/venue/` 相关页面。

## 主要功能

- 总开关：控制所有自动流程。关闭时不会接管手动操作。
- 校区 / 场馆 / 运动项目：选择目标预约项目，脚本会自动推导对应预约页。
- 场地 / 时段：选择期望的场地号和时间段；不选则表示不限。
- 触发时间：设置定时开始轮询的时间，例如 `07:59:58`。
- 轮询间隔：设置扫描频率，单位为毫秒。
- 最长轮询：达到设定分钟数后自动停止。
- 干跑模式：只扫描和记录命中结果，不自动点击场地。
- 主动刷新：通过预约接口刷新日期数据，辅助发现空位。
- 命中重载：接口发现空位但页面表格未更新时，自动重载预约页。
- 自动选同伴：提交前按设置选择同伴。
- 自动验证码：支持人工验证、本地自动、API 验证三种模式。
- 自动点支付：订单生成后尝试点击支付按钮。
- 随机候选：多个命中场地时随机选择一个。
- 命中后停止：命中并点击候选后停止轮询。
- 停止：强制停止轮询、流程推进、验证码处理和支付重试。
- 折叠面板：将控制卡片收起为右下角圆形按钮。



## 验证码模式

- 人工验证
- 本地自动
- API 验证


## 使用建议

1. 首次使用保持干跑模式开启。
2. 选择校区、场馆、项目、场地和时段。
3. 点击扫描一次，确认日志中的命中结果符合预期。
4. 打开总开关。
5. 使用定时启动或立即轮询。
6. 确认流程稳定后，再关闭干跑模式。
